## 1. Architecture in one breath

A **Google Sheet is the single source of truth.** A **single-file HTML dashboard** (`index.html`) is a read/write *view* of it; it holds no authoritative state. A **Google Apps Script web app** (`apps_script.gs`) is the only thing that touches the sheet: `doGet` serves reads, `doPost` serves writes, both returning JSON. The dashboard talks to the deployed Web App URL (`SCRIPT_URL`) via `fetch`. `SCRIPT_URL` is loaded from a **gitignored `config.js`** (`window.CONFIG.SCRIPT_URL`); a saved URL in `localStorage['fin_url']` overrides it, and when neither is present the boot path shows the setup screen.

```
Google Sheet  в—„в”Җв”Җв–ә  apps_script.gs (Web App: doGet reads / doPost writes)  в—„в”Җв”Җfetchв”Җв”Җв–ә  index.html (browser)
   (truth)            tests.gs В· debug.gs В· migrate.gs live in same Apps Script project
```

**Source of truth for code + docs: the public GitHub repo `bencoscia/finance-dashboard`.** `index.html` is served from **GitHub Pages** off that repo. The backend lives in a separate Apps Script project; the repo's `.gs` files are reference copies, not the live backend. Claude reads the live files via `raw.githubusercontent.com` (CDN-cached ~5 min); pushes are manual вҖ” you commit. `config.js` (the `SCRIPT_URL` bearer secret) is gitignored and never in the repo.

## 3. Invariants вҖ” do not violate

1. **The Sheet is truth.** The dashboard never invents state; it reads from / writes to the Sheet. Client caches (`allData`, `nwData`, `loanData`, `mortgageData`) are views and must be invalidated when a write changes them.
2. **Zero inline `on*=` attributes in HTML.** No `onclick="вҖҰ"` (or any `onвҖҰ=`) in rendered markup. Interactions go through `data-action`/`data-args` + the document-level `ACTION_MAP` dispatcher, or through `.onclick`/`.oninput` **property closures**. The JS-string-inside-HTML-attribute escaping bug class caused three outages and is now structurally extinct вҖ” do not reintroduce it. (`.onclick`/`.oninput` property assignments are *allowed* вҖ” real closures, never the hazard; audit.md's count of "seven" is stale, it's ~12 now *(verify)*.)
3. **Dual-router rule.** A read action must be registered in **both** dispatchers in `apps_script.gs`: the `doGet` chain (~L137) **and** the unified action chain (~L93). A write goes in `doPost` (~L146) and, if it mutates money or rows, is wrapped in `_withIdem`.
4. **No silent staleness.** Any value interpolated into already-rendered HTML must be re-rendered by every write that changes it; cross-tab caches affecting net worth must be nulled on relevant writes.
5. **Protan-safe visuals (Ben is protanopic).** Never encode meaning by hue alone. Separate series by **lightness, dash pattern, axis, shape, and a text label** вҖ” at least two non-color channels, always.
6. **`.gs` is ES5 + ASCII only.** `var`/`function(){}`, no arrow fns, no template literals, no non-ASCII characters. `index.html` JS follows the same `var`/`function`/string-concatenation style; no template literals in rendered HTML strings.
7. **Chart.js discipline.** Destroy the instance before re-rendering (`if(cX){cX.destroy();cX=null;}`). There is **no time-scale adapter** loaded вҖ” use category-string labels + `maxTicksLimit`. Chart.js cannot read CSS custom properties вҖ” resolve them with `getComputedStyle` (helper: `cssv(name, fallback)`).
8. **Bump cache versions on shape change.** Change a cached payload's fields вҶ’ bump the matching `CACHE_VERSION` (`monthly_v3`) or `CARD_CACHE_VERSION` (`cards_v1`).

## 4. Traps / failure modes (each has bitten before)

- **Deployment skew.** "New deployment" mints a *new URL that serves the old code forever*. Always **pencil-edit the existing deployment вҶ’ New version** (same URL). Symptom of a stale deploy: `Unknown action: <x>` from a read/write you just added.
- **Dual-router drift.** Forgetting to add a read to *both* chains (Invariant 3) вҶ’ works via one entry point, fails via the other.
- **Interpolated-UI staleness.** A write updates the Sheet but not the baked-in HTML/caches вҶ’ the tab shows old numbers until reload.
- **`--blue` is green.** `--blue` = `--green` = `#1E5C46`. The actual blue is `--c2` `#3E6B8F`. Misnamed token вҖ” read В§6 before picking colors.
- **Mortgage balance вҖ” one source (resolved 2026-06-16).** The `Loans!MвҖ“W` ledger is the source of truth; `Physical Assets!B8` is now a **formula** mirroring its current balance вҖ” `=XLOOKUP(TRUE, Loans!N8:N400, Loans!O8:O400, "NO PAID ROW", 0, -1)` (balance on the last `TRUE`-flagged row). Net worth reads B8, the mortgage tab reads the ledger, and they can't drift because B8 derives from the ledger. The one manual input is the **col-N paid flag** вҖ” tick it the month a payment posts. **Do not overtype B8 with a literal** вҖ” that re-creates the old hand-maintained duplicate (the ~$343 / one-month drift). Two Sheets gotchas: the Excel `LOOKUP(2,1/(вҖҰ))` last-match idiom returns `#N/A` here вҖ” use XLOOKUP; and bound the range to the scheduleвҖҷs last row so a stray `TRUE` in another loan block below canвҖҷt be matched (XLOOKUP `-1` searches bottom-up). If B8 ever reads `NO PAID ROW`, net worth silently books $0 mortgage.
- **`config.js` isn't served on Pages.** It's gitignored, so on GitHub Pages `window.CONFIG` is undefined and `SCRIPT_URL` is empty вҖ” by design. The setup screen is the Pages onboarding (paste the URL once вҶ’ saved to that browser's `localStorage['fin_url']`). `bootDashboard()` **must** fall back to `showSetup()` when there's no URL; a missing `else` there once left Pages blank-white. Works locally because `config.js` is present.
- **No client-side password gate.** One was added and removed вҖ” it's theater: page source (hence any hash) is world-readable on Pages, and `SCRIPT_URL`, not the gate, protects the data. Don't reintroduce. Real access control = `SCRIPT_URL` secrecy now, a server-validated token later (В§9).
- **`SCRIPT_URL` precedence:** `localStorage['fin_url']` (set via setup) wins over `config.js`. A stale saved URL silently overrides a fresh `config.js` вҖ” clear it via the setup screen if you repoint.
- **HSA double-reimbursement (cardinal risk).** Paying out a receipt twice is an over-withdrawal the IRS can claw back. Two layers: `reimburseReceipt` is `_withIdem`-wrapped **and** writes `reimbursed_amount` as an **absolute** value (a retry writes the same number, never adds), plus a server `_reimburseGuard` rejecting `> amount`, negative, `funding=HSA`, and pre-establishment rows. The UI "Reimburse to total ($)" field is the absolute total, not an increment вҖ” keep it so.
- **HSA establishment gate.** Only expenses on/after `Config!hsa_established` qualify; earlier rows are flagged вҡ  and excluded from every total. **Blank `hsa_established` вҶ’ gate OFF** (nothing disqualified) and the tab shows a вҡ  banner вҖ” set it or totals overstate.
- **HSA balance is `B1`, blank вү  0.** `HSA Receipts!B1` (manual) is the only balance source; **blank вҶ’ `null` вҶ’ renders "вҖ”", never `$0`**, and `reimbursableNow`/`strandedEntitlement` stay `null` (not computed against 0). HSA is **not** in net worth yet (В§9). Receipts store a **Drive link**, never an embedded image.
- **`cfg()` cache is shape-versioned (`CFG_CACHE_KEY`, 5-min TTL) вҖ” bump it on every Config-shape change, not just `getNetWorth`'s.** Adding `hsaEstablished` without bumping `cfg_v1`вҶ’`cfg_v2` meant a config cached moments before the redeploy kept serving the old shape (missing the field) for up to 5 min вҖ” surfaced as "No HSA establishment date set" even with the sheet row correct. Same root cause as the documented `nw_v1`вҶ’`nw_v3` incident (В§ bug class 2), now also true of `cfg()`. (Bumped again to `cfg_v3` for `hsaReceiptFolder`, then **`cfg_v4`** for `hsaDeductible`/`hsaContribLimit`/`hsaDeductibleResetMonth`.) `testConfigParsing` asserts the round-trips, but it can't catch a forgotten version bump вҖ” that's a manual checklist item.
- **`HSA_CACHE_VERSION`** has climbed `hsa_v1вҶ’v5` (v2 `needsReview`, v3 `planYear`, v4 `contribRows`, **v5 date-aware balance** = `inflowsSince`/`outflowsSince` + `computedBalance` now projects B1 forward). Any cached-payload shape change needs the bump or stale reads persist up to the TTL.
- **Balance reconciliation is done AS OF the B1 anchor date (`balanceAsOf`/B2), not "now."** Earlier it compared an all-time reconstruction against a possibly-stale B1, so spending logged *after* B2 inflated `drift` вҖ” a $345 drift that was really $312 of post-B2 card spend + $32 interest. Fix: `_hsaPlanYear` splits inflows/outflows by B2, reconciles `computedThruB2` vs B1 (drift вүҲ unlogged interest only), and the displayed balance projects B1 forward by since-B2 activity (`computedBalance = B1 + inflowsSince вҲ’ outflowsSince`). **Keep B2 current when you update B1**, or the "since" projection grows. Card spend counts as outflow even for `needs_review` drafts (cash left regardless of qualification); the deductible total excludes drafts.
- **Contributions-reader gotcha (drift вүҲ total contributions).** If the reconciled balance's drift comes out вүҲ your lifetime contributions, `inflows` is landing at $0: the `HSA Contributions` sheet is found but every row is skipped. Algebra: `B1 = contrib + interest вҲ’ spend`, so with `inflows=0`, `drift = B1 + spend = contrib + interest`. `_hsaReadContributions` now locates the **amount** column by header name (robust to column reorder) and strips `$`/commas (text-formatted amounts), and `planYear.contribRows` lets the UI show "sheet found, 0 amounts read" instead of a garbage negative balance.
- **The plan-year balance is a reconciliation, not a source of truth.** Fidelity (`B1`) is authoritative; `_hsaPlanYear` reconstructs a balance from logged inflows (the `HSA Contributions` sheet) minus outflows (card spend + reimbursements) and reports `drift = B1 вҲ’ computed`. Small drift (вүӨ max($50, 2%)) is expected and вүҲ unlogged FDRXX interest (interest is deliberately **not** logged, to keep the contributions sheet clean); large drift means activity is unlogged вҖ” re-import the Fidelity export. **Card spend counts as a balance outflow even for `needs_review` drafts** (cash left the account regardless of qualification), whereas the *deductible* total excludes drafts вҖ” the two sums intentionally differ. Interest, if ever logged, is a balance inflow but does **not** count toward the contribution limit (limit gauge = employee+employer only).
- **`needs_review` drafts are deliberately invisible to totals.** A scanned-but-malformed receipt is present (and visible, sorted to top with вңҺ) but contributes **$0** to `unreimbursed`/`totalSubstantiated`/the pool until you confirm it via Edit. Intentional (don't trust unverified data) вҖ” but it means "I scanned it and the pool didn't move" is expected, not a bug.
- **The HSA projection is a two-engine, nominal-dollar model.** Engine 1 (recurring tax avoided = spendГ—(tc+f)) is certain and market-independent; Engine 2 (the hoard) is the only market-dependent part and only matters on receipts you *keep* (don't reimburse). Don't conflate them, and don't read the old "$8k/yr for 30 yrs, hold forever" framing back in вҖ” that was ~80Г— too high because it used the contribution limit instead of the real claim rate and never reimbursed. `hsaProjCompute` is pure and cross-checked offline; at the demo inputs ($2,800/$500/20/30) it must produce Engine 1 $1,054/yr and Engine 2 HSA $43,145 / IRA $31,078 / taxable $22,469. Spend/hoard auto-fill from the ledger TTM; a user override (saved in `localStorage['hsa_calc_v2']`) wins over the TTM prefill.
- **Folder scan: filename is load-bearing data.** `scanHsaFolder` parses `DATE~provider~amount.pdf` into the row's date/provider/amount вҖ” a filename typo becomes a wrong ledger entry. `_parseReceiptName` is strict and **skips + reports** anything off-pattern rather than guessing. Delimiter is `~` (provider may contain spaces/underscores; it may **not** contain `~`).
- **Folder scan dedups on Drive `fileId` (col L), not on content** вҖ” so a receipt entered *both* manually and via the scan creates **two** rows (the manual one has no fileId to match). Pick one path per receipt; the scan's created/skipped report makes a surprise duplicate visible immediately. Auto-created rows are `funding=OOP` (safe default вҖ” surfaces as a reimbursable claim) with blank category/description to fill in.
- **`DriveApp` needs the Drive OAuth scope.** Adding the folder scan introduces a new scope; the first `scanHsaFolder` run prompts re-authorization. After deploying, run it once from the editor (or via the button) and approve, or the web app returns an auth error.

**Read actions (17)** вҖ” registered in both chains, served by `doGet`:
`all В· quick В· monthly В· networth В· transactions В· fixed В· variableEntries В· accounts В· cardTotals В· cardBalances В· cardTxns В· config В· loans В· mortgage В· hsa В· ledgerMonthly В· ledgerTxns`

**Write actions (23)** вҖ” `doPost` (вҳ… = `_withIdem`-wrapped):
`addTransactionвҳ… В· updateTransaction В· deleteTransaction В· splitTransaction В· setFixedPaid В· updateFixedCost В· updateFixedName В· updateLoanBalanceвҳ… В· makeCardPaymentвҳ… В· voidLastPaymentвҳ… В· setSeedBalance В· addVariableEntryвҳ… В· updateVariableEntry В· deleteVariableEntry В· makeCheckingEntryвҳ… В· saveNetWorthSnapshotвҳ… В· logCreditSnapshotвҳ… В· addMonth В· reimburseReceiptвҳ… В· addHsaReceiptвҳ… В· scanHsaFolder В· updateHsaReceiptвҳ… В· ledgerAddTxnвҳ…`

`voidLastPaymentвҳ…` matters most of the 2026-07 additions: it voids the *most recent* Card Payments row, so a manual retry after a dropped response would void a **second, older payment** silently вҖ” the same failure class as the documented double-payment incident. The idemKey makes the auto-retry safe and the cached result makes an immediate manual re-click a no-op. `addMonth` is deliberately not wrapped вҖ” it's naturally idempotent (errors if the sheet already exists).

`scanHsaFolder` is **not** `_withIdem`-wrapped вҖ” it's idempotent by construction (dedup on Drive `fileId` stored in `HSA Receipts` col L), takes no args (so a future time-trigger can call it directly), and serializes via `LockService`. Reads the Drive folder named in `Config!hsa_receipt_folder` (requires the Drive OAuth scope вҖ” first run prompts re-authorization). A malformed filename is imported as a **draft** (`needs_review`, col M) via best-effort parse rather than skipped; only a name with neither a date nor an amount is skipped. `updateHsaReceipt` is the general edit path (fix any field; clears `needs_review`); reimbursed amount/date stay on `reimburseReceipt` so that remains the single money-moving path.

**Sheets touched:** `Loans`, `Physical Assets`, `Credit`, `Discover Savings`, `Portfolio Management`, `Net Worth`, `Card Payments`, `Card Trackers`, `Config`, `HSA Receipts`, `Template`, and the monthly sheets (`"<Month> YYYY"`).

**Key helpers:** `readSheet(name|sheet)` (cached per-request 2-D values) В· `_withIdem(payload, fn)` (idempotency keys in CacheService, 6 h) В· `cfg()` (reads `Config`, 5-min cache under shape-versioned `CFG_CACHE_KEY` вҖ” bump on any shape change, see В§4 trap) В· cache invalidation per month/card/net-worth/HSA.

## 7. Deployment workflow

**Frontend (`index.html`, CSS, docs):** commit + push вҶ’ **GitHub Pages** serves it. `raw.githubusercontent.com` and Pages are CDN-cached ~5 min вҖ” hard-refresh after a push. `config.js` is **never committed** (local-only; on Pages the setup screen collects the URL into localStorage).

**Backend (`apps_script.gs`):** edit in the Apps Script editor, then **Deploy вҶ’ Manage deployments вҶ’ pencil-edit the existing deployment вҶ’ Version: New version вҶ’ Deploy.** Never "New deployment" (new URL = stale code). Commit the updated `.gs` so the repo's reference copy doesn't drift. Smoke test: open the affected tab; `Unknown action: <x>` means the deploy didn't take.

## 8. Spreadsheet schemas that matter

- **`Loans!A4:J25` вҖ” student loans.** A origNum, B renum, C type, **D balance (edit target for `updateLoanBalance`)**, E rate, F interest, G principal, H payment due, I new balance, J % of balance. Row 26 totals (D26 balance, F26 interest); H27 avg rate; H28 % grad school.
- **`Loans!MвҖ“W` вҖ” mortgage amortization (full schedule to payoff Feb 2055).** Header: N3 original rate, N4 valuation (`='Physical Assets'!B6`), N5 purchase. Ledger rows 8+: M date, N description/paid-flag (`TRUE`=logged actual), O balance, P payment (P&I + escrow), Q interest, R PMI, S property tax, T principal, U equity %. **Row 40 = `Refinance` marker** (its O cell = new rate; interest switches N3вҶ’O40 after it). Cost box: W9 initial projected P&I, W10 current projected P&I (`=SUM(Q:Q)+SUM(T:T)`), W11 savings (`=W9-W10`). Read-only in the dashboard via `getMortgageData`. `Physical Assets!B8` mirrors this ledger's current balance (last `TRUE`-flagged row's O) via formula, and is what `_computeNetWorth` reads for the mortgage liability.
- **`Physical Assets`.** B3вҖ“B5 home estimates (Redfin/Zillow/realtor), B6 avg valuation, **B8 mortgage balance вҖ” formula `=XLOOKUP(TRUE, Loans!N8:N400, Loans!O8:O400, "NO PAID ROW", 0, -1)` deriving the ledger's current-month balance; net-worth source (was hand-typed, now self-maintaining вҖ” don't overtype)**, B9/B10 equity $ / %.
- **Monthly sheets `"<Month> YYYY"`.** Per-month transactions/fixed/variable; the hot path for `getMonthlyData` (cached: CacheService + durable PropertiesService tier; recompute only changed months).
- **`Config`.** `key | value | notes`; read by `cfg()`. Holds `BUDGET`, `DC_AMOUNT`, `DC_CLOSED_MONDAYS`, `QUARTERLY_EXPENSES`, `PROJECTION_TARGET`, `SEED_MONTH_CONFIG` (each with a hardcoded fallback), and `hsa_established` (HSA qualifying-date gate; **no fallback вҖ” blank disables the gate**, see В§4), `hsa_receipt_folder` (Drive folder ID the receipt scan reads), `hsa_deductible` (plan-year deductible $; blank вҶ’ tracker hidden), `hsa_contrib_limit` (annual contribution limit $; blank вҶ’ gauge hidden), `hsa_deductible_reset_month` (1вҖ“12, default 1 = calendar year; lets a mid-year plan year be a one-cell change).
- **`HSA Contributions` вҖ” contribution/inflow log (optional).** `date | amount | type | notes`; header row 1, data **rows 2+**. `type` вҲҲ `employee`|`employer`|`interest`. Read by `_hsaReadContributions(ss)` вҶ’ `null` when the sheet is absent (balance/contrib gauges then hide; the deductible tracker still works off receipts alone). Populated by pasting the Fidelity-export backfill and repopulated by re-importing going forward. Feeds `_hsaPlanYear`: contributions+interest are balance inflows; only employee+employer count toward the contribution limit. Kept separate from `HSA Receipts` so that sheet stays a pure expense ledger. Pure metrics tested by `testHsaPlanYear`.
- **Ledger sheets (migration target, new 2026-07)** -- four flat tables, headers row 1, data rows 2+, rows located by stable `id` (col A), appends only, no formulas below the data block. `Txns`: `id | date | month | description | amount | method | category | discretionary | ben | jenna` (`category` in `onetime|groceries|gas|transfer`; `month` is an explicit YYYY-MM assignment -- a July-1 charge can belong to June's books). `Fixed Log`: `id | month | name | source | amount | paid | paid_date`. `Income Log`: `id | month | date | source | amount`. `Variable History`: `month | category | amount` -- **display/TTM-estimation data only, never summed into totals.** THE AGGREGATION RULE (`_ledgerAggregate` in `ledger.gs`, enforced there and only there): every dollar counts exactly once -- txns via their row, fixed via the paid flag; `transfers` (category `transfer` OR any Checking row with amount < 0) are money movement, excluded from both expenses and income; card negatives (cash back/refunds) net against spend; `totalExpenses = (txn sum - transfers) + fixedPaid`. Created by `setupLedgerSheets()`; populated by importing the migration CSVs (regenerate from a fresh xlsx export via `extract_monthly.py` at cutover). `LEDGER_CACHE_VERSION` (`ledger_v1`) -- bump on `getLedgerMonthly` shape change.
- **`Net Worth`.** Snapshots written by `saveNetWorthSnapshot`.
- **`HSA Receipts` вҖ” reimbursement-entitlement ledger.** Metadata: **`B1` = current HSA balance (manual; blankвҶ’`null`, never 0), `B2` = balance as-of date.** Headers on **row 3**, data **rows 4+**. Cols AвҖ“M: A `id` (stable positive int, never reused вҖ” `reimburseReceipt`/`updateHsaReceipt` locate rows by id, not position), B `date_incurred`, C `amount`, D `provider`, E `description`, F `category`, G `funding` (`OOP`|`HSA`), H `receipt_link` (Drive URL), I `reimbursed_amount` (**absolute**; written by `reimburseReceipt`), J `reimbursed_date`, K `notes`, L `source_file_id` (Drive fileId when created by the folder scan; blank for manual adds вҖ” the dedup key), **M `needs_review` (`TRUE` for a malformed-import draft; blank once confirmed)**. Read by `getHsa` вҶ’ `{established, hsaBalance, balanceAsOf, unreimbursed, reimbursableNow, totalSubstantiated, strandedEntitlement, planYear, rows[]}` where each row carries `needsReview` (col L is internal, not exposed) and `planYear` carries the deductible/contribution/reconciled-balance metrics from `_hsaPlanYear` (see В§4). **A `needs_review` draft is excluded from every total until confirmed** (it's unverified data вҖ” `_hsaRollups` gives it `status:'needs_review'` and skips it from `unreimbursed`/`totalSubstantiated`); it sorts to the top of the ledger and shows the вңҺ glyph. Written by `addHsaReceipt` (plain append вҖ” no formulas below the data block; **id = max existing id + 1 inside a `LockService.getScriptLock()`**; `needsReview:true` relaxes validation so a partial draft can be stored), `reimburseReceipt`, `scanHsaFolder`, and **`updateHsaReceipt`** (the general edit/correction path вҖ” validates strictly, writes BвҖ“H, and clears col M; reimbursed amount/date are deliberately **not** editable here so `reimburseReceipt` stays the single money-moving path). Folder scan imports PDFs from `Config!hsa_receipt_folder` named **`DATE~provider~amount.pdf`** (delimiter `~`; `_parseReceiptName` is strict). **A name that fails strict parse is salvaged by `_bestEffortReceipt`** (tolerates `~` or `_`, pulls a calendar-valid date via `_isRealDate` and a positive amount, rest вҶ’ provider) and imported as a `needs_review` draft; only a name with **neither** a date nor an amount is skipped + reported. Pure helpers `_hsaRollups` / `_reimburseGuard` / `_parseReceiptName` / `_bestEffortReceipt` / `_isRealDate` are unit-tested (`testHsa`, `testParseReceiptName`, `testBestEffortReceipt`); `addHsaReceipt` + `updateHsaReceipt` have scratch-sheet integration tests (`testAddHsaReceipt`, `testUpdateHsaReceipt`, sheet `_TestScratchHsa_` вҖ” never the live sheet). **Correct any row in-dashboard via the edit modal (вңҺ Edit on every row); the Sheet remains a valid fallback.**
